# 대시보드 시각화 기능 QA 시나리오

## 📝 개요

이 QA 시나리오는 인터랙티브 세션 브라우저의 히트맵 시각화 기능이 정상적으로 구현되었는지 검증합니다. GitHub 스타일 히트맵을 통한 세션 활동 패턴 시각화의 품질과 사용성을 평가합니다.

## 🎯 테스트 목적

1. **히트맵 컴포넌트** 렌더링 정확성 확인
2. **실시간 데이터 업데이트** 기능 검증
3. **사용자 인터랙션** 반응성 테스트
4. **성능 기준** 충족 여부 확인

## 📋 QA 시나리오

### Scenario 1: 히트맵 컴포넌트 기본 기능 검증

**목적**: 히트맵이 정확한 데이터로 렌더링되는지 확인

**전제 조건**:
- Tauri 대시보드가 실행 중이어야 함
- 최소 1개 이상의 tmux 세션이 활성화되어 있어야 함

**테스트 단계**:
1. **대시보드 접근 및 기본 렌더링**
   ```bash
   cd /home/archmagece/myopen/scripton/yesman-claude
   uv run ./yesman.py dashboard --dev
   ```
   - **기대 결과**: 대시보드가 에러 없이 실행되어야 함
   - **검증 포인트**: 
     - 브라우저에서 localhost:3000 접근 가능
     - 히트맵 영역이 표시됨
     - 로딩 인디케이터가 5초 이내 사라짐

2. **히트맵 데이터 구조 검증**
   - **수동 확인**: 브라우저 개발자 도구에서 네트워크 탭 확인
   - **기대 결과**: `/api/dashboard/heatmap` 엔드포인트 정상 응답
   - **검증 포인트**:
     - 응답 시간 < 5초
     - JSON 데이터 구조가 24시간 x 7일 형태
     - 각 셀에 활성도 수치가 포함됨

3. **시각적 히트맵 표시 확인**
   - **수동 확인**: 히트맵 그리드가 정상 표시되는지 확인
   - **기대 결과**: GitHub 스타일 히트맵 시각화
   - **검증 포인트**:
     - 24시간 x 7일 그리드 구조
     - 활성도에 따른 색상 그라데이션 (초록→노랑→빨강)
     - 요일/시간 라벨 정확히 표시
     - 반응형 레이아웃 (모바일/데스크톱)

### Scenario 2: 실시간 업데이트 기능 검증

**목적**: 히트맵이 실시간으로 데이터를 갱신하는지 확인

**테스트 단계**:
1. **자동 업데이트 주기 확인**
   - **수동 확인**: 대시보드를 5분간 관찰
   - **기대 결과**: 5초 간격으로 데이터 갱신
   - **검증 포인트**:
     - 네트워크 탭에서 API 호출 주기 확인
     - 히트맵 색상 변화 감지
     - 메모리 누수 없이 안정적 동작

2. **실제 세션 활동 반영 테스트**
   ```bash
   # 새로운 tmux 세션 생성하여 활동 발생
   tmux new-session -d -s test-activity
   tmux send-keys -t test-activity 'echo "Activity test"' C-m
   tmux send-keys -t test-activity 'ls -la' C-m
   ```
   - **기대 결과**: 새로운 활동이 히트맵에 반영됨
   - **검증 포인트**:
     - 5초 이내 히트맵 업데이트 확인
     - 해당 시간대 셀의 색상 변화
     - 활성도 수치 증가

### Scenario 3: 사용자 인터랙션 검증

**목적**: 히트맵의 인터랙티브 기능이 정상 작동하는지 확인

**테스트 단계**:
1. **마우스 오버 툴팁 기능**
   - **수동 확인**: 히트맵 셀에 마우스 커서 이동
   - **기대 결과**: 상세 활동 정보 툴팁 표시
   - **검증 포인트**:
     - 툴팁이 즉시 나타남 (< 200ms)
     - 정확한 시간/날짜 정보 표시
     - 활동 수치 및 세션 정보 포함
     - 툴팁이 화면 영역을 벗어나지 않음

2. **세션 필터링 기능**
   - **수동 확인**: 다중 세션 선택 UI 테스트
   - **기대 결과**: 선택된 세션의 통합 히트맵 표시
   - **검증 포인트**:
     - 세션 목록이 정확히 표시됨
     - 체크박스/드롭다운 인터랙션 정상
     - 필터 적용 시 히트맵 즉시 업데이트
     - "모든 세션" 옵션 정상 작동

3. **데이터 내보내기 기능**
   - **수동 확인**: 내보내기 버튼 클릭
   - **기대 결과**: PNG/SVG 파일 다운로드
   - **검증 포인트**:
     - 파일 다운로드 성공
     - 이미지 품질 적절함
     - 파일 크기 합리적 (< 1MB)

### Scenario 4: 성능 및 접근성 검증

**목적**: 히트맵이 성능 기준을 충족하고 접근성을 고려했는지 확인

**테스트 단계**:
1. **로딩 성능 측정**
   ```bash
   # Chrome DevTools Performance 탭 또는 Lighthouse 사용
   # 또는 간단한 네트워크 시간 측정
   curl -w "@curl-format.txt" -o /dev/null -s "http://localhost:3000/api/dashboard/heatmap"
   ```
   - **기대 결과**: 5초 이내 완전 로딩
   - **검증 포인트**:
     - 첫 번째 바이트까지의 시간 (TTFB) < 1초
     - DOM 완전 로딩 시간 < 3초
     - 히트맵 렌더링 완료 < 5초

2. **메모리 사용량 모니터링**
   - **수동 확인**: 브라우저 작업 관리자에서 메모리 사용량 확인
   - **기대 결과**: 안정적인 메모리 사용 패턴
   - **검증 포인트**:
     - 초기 메모리 사용량 < 100MB
     - 1시간 사용 후 메모리 증가 < 50MB
     - 메모리 누수 없음 (GC 후 원래 수준 복구)

3. **다양한 환경 호환성**
   - **테스트 환경**: Chrome, Firefox, Safari (가능한 경우)
   - **기대 결과**: 모든 주요 브라우저에서 정상 동작
   - **검증 포인트**:
     - 히트맵 렌더링 일관성
     - 인터랙션 기능 동일
     - 성능 차이 최소화

### Scenario 5: 오류 상황 처리 검증

**목적**: 예외 상황에서 적절한 오류 처리가 되는지 확인

**테스트 단계**:
1. **네트워크 오류 시뮬레이션**
   ```bash
   # API 서버 임시 중단
   pkill -f "yesman.py dashboard"
   ```
   - **기대 결과**: 적절한 오류 메시지 표시
   - **검증 포인트**:
     - "연결 실패" 메시지 표시
     - 자동 재연결 시도
     - 이전 데이터 유지 (가능한 경우)

2. **빈 데이터 상황 처리**
   ```bash
   # 모든 tmux 세션 종료
   tmux kill-server
   ```
   - **기대 결과**: 빈 히트맵 상태 표시
   - **검증 포인트**:
     - "활동 없음" 메시지 표시
     - 빈 그리드 구조 유지
     - 새로운 세션 생성 시 자동 업데이트

## 📊 성공 기준

### 정량적 기준
- [ ] **로딩 시간**: 5초 이내 완전 로딩
- [ ] **업데이트 주기**: 5초 간격 정확 실행
- [ ] **메모리 사용량**: 100MB 이하 안정적 유지
- [ ] **응답 시간**: 툴팁 표시 200ms 이내
- [ ] **데이터 정확성**: 실제 활동과 100% 일치

### 정성적 기준
- [ ] **시각적 품질**: GitHub 스타일 히트맵 구현
- [ ] **사용자 경험**: 직관적이고 반응적인 인터랙션
- [ ] **접근성**: 색맹 사용자 고려 (대안 표시 방법)
- [ ] **반응형 디자인**: 모바일/태블릿/데스크톱 지원
- [ ] **오류 처리**: 적절한 피드백 및 복구 메커니즘

## ⚠️ 알려진 제한사항

1. **브라우저 호환성**: IE11 이하 버전 미지원
2. **데이터 보존**: 브라우저 새로고침 시 임시 데이터 손실
3. **성능 한계**: 매우 많은 세션(50개+) 시 성능 저하 가능
4. **시간대 처리**: 로컬 시간대만 지원, UTC 변환 미지원
